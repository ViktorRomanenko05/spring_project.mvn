Мини - приложение "Аэропорт"
Сущности:
         Passenger;
         Employee; - наследуют общий абстрактный класс Employee
                     и переопределяют метод регистрации на рейс

         Ticket;
         Plane;

         Flight; - содержит методы для управления процессом подготовки
                   рейса и полётом;

Классы - менеджеры:
         Office - хранит данные сотрудников и управляет ими;
         TicketOffice - управляет данными билетов и их продажей;
         WaitingRoom - хранит данные пассажиров и управляет ими;

Сценарий класса AirportClient:

         В целях упрощения задачи для демонстрации были созданы один рейс и для
него один самолет.

         1) Список сотрудников хранится в файле и считывается при запуске приложения.
            Каждый сотрудник имеет роль (pilot, stewart, techniker, sakes agent).

         2) Список пассажиров (500 элементов) генерируется с помощью библиотеки Faker
            (То есть список персонала статичен, данные пассажиров разные при каждом запуске).

         3) Каждому пассажиру произвольно выбранным сотрудником отдела продаж продается билет
            на произвольное направление, выбранное из
            доступных в enum - Destination. При этом билету устанавливается статус
            "confirmed" и билет сохраняется в статичном хранилище класса TicketOffice
            и в хранилище конкретного объекта Passenger.

         4) На борт из списка сотрудников выбираются пилоты и стюарты из
            списка, имеющие статус FREE. Статус при этом изменяется на BUSY. Количество
            членов экипажа определяется требованиями указанными в объекте Plane.
            Если необходимое количество членов экипажа со статусом FREE не найдено -
            регистрация на рейс останавливается и вылет не осуществляется.

         5) Из зала ожидания выбираются все пассажиры с билетом на нужный рейс
            (в нашем случае - New York), при этом данные пассажиров удаляются из хранилища зала
            ожидания и сохраняются в хранилище рейса. Их билеты получают статус USED

         6) Если количество членов экипажа достаточное и количество пассажиров - больше 0 -
            самолет выполняет рейс.
            При этом в консоли можно увидеть данные о самолёте и количество пассажиров на борту.
            После посадки хранилища данных экипажа и пассажиров в объекте рейс - очищаются,
            члены экипажа получают статус free


         В задаче согласно заданию продемонстрировано взаимодействие объектов разных классов
         с использованием фреймворка spring.
